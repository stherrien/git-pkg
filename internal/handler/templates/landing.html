git-pkg.dev

Turn GitHub Releases into pip-installable Python packages. Zero accounts. Zero uploads. Zero config.

What is this?

A stateless proxy that translates GitHub Releases into a PEP 503 package index. You tag a release, attach a .whl file, and your users can pip install it. git-pkg.dev never stores your code or credentials — it just translates the GitHub API into the format pip expects, and pip downloads directly from GitHub's CDN.

For package authors

1
Add a GitHub Action to build your .whl on tagged releases.
2
Tag a release: git tag v1.0.0 && git push origin v1.0.0
3
That's it. No registry account. No API tokens. No upload step.

Install a public package

pip install mypackage --index-url https://git-pkg.dev/owner/

Install from a private repo

pip install mypackage \
  --index-url https://x:${GH_TOKEN}@git-pkg.dev/myorg/

Pin in requirements.txt

--index-url https://git-pkg.dev/myorg/
mypackage==1.2.3
another-pkg>=2.0

Why not just pip install git+https://?

×
No version resolution — installs HEAD, not a pinned release.
×
Clones the repo and builds from source every time. Slow.
×
pip freeze outputs a git hash, not a version number.
✓
git-pkg.dev gives you real PEP 440 versions, pre-built wheels, and standard pip semantics.
